# 提示词模板库使用说明

这个模板库不是“装饰性提示词”，而是用于把任务输入变成标准工单。

使用方式：

1. 先选最接近当前任务的模板；
2. 把模板中的【占位内容】替换成自己的材料；
3. 不要一次塞入多个互相无关的任务；
4. 如果任务复杂，先用“复杂任务拆解模板”；
5. 如果项目已经讨论很多轮，先用“项目复盘模板”收束；
6. 如果要让代码模型修改项目，优先用“Codex 强指令模板”。

通用原则：

- 让模型先判断，再解释；
- 让模型区分确定结论、合理推测、风险点和未知项；
- 让模型给最小可执行方案，而不是泛泛建议；
- 让模型输出可验证步骤，而不是只讲道理；
- 对代码、环境、仿真任务，必须要求运行命令和预期输出。
